# Home Quick View Design

## Purpose

Homeの要約カードから、いきなり別ワークスペースへ移動しない導線を作る。操作階層は「Homeの要約」「Home上の軽量詳細」「全情報の専用画面」の3段階にする。

## Approved behavior

- 「作業一覧を見る」はHome上で稼働中作業のオーバーレイを開く。
- 作業オーバーレイ内の「記録ですべて見る」だけが記録ワークスペースへ移動する。
- ユーザータスク見出しの拡大操作はHome上で現在の未処理項目をオーバーレイ表示する。
- ユーザータスクオーバーレイ内の「ユーザータスクですべて見る」だけがユーザータスクワークスペースへ移動する。
- Homeカード内の「質問」「承認」「確認」「手動作業」は、従来どおり該当種類で絞り込んだユーザータスクワークスペースへ直接移動する。
- Homeカード内の個別タスク・個別ユーザータスクの既存操作は変更しない。

## Implementation boundary

既存の`OverlayFrame`と`NowListOverlay`を再利用する。新規画面は現在のユーザータスクを読む軽量オーバーレイ1つだけとし、データ契約やElectron境界は変更しない。

## Verification boundary

対象導線のコンポーネントテストとアプリ統合テスト、TypeScriptを含む通常ビルドだけを実行する。長時間の負荷試験や既に確認済みのクリック耐久試験は繰り返さない。
